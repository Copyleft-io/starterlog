Application Structure | Detail

Our application directory structure is very simple and straight-forward. Here is a quick overview that provides some additional detail.


.git/
# Git Project Directory
# The .git/ folder contain all the information that is necessary for managing our project in git version control.


.tmp/
# Temporary Staging Directory
# The .tmp/ folder is the temporary staging directory where our grunt build process prepares and packages our application as defined in Gruntfile.js


app/
# Source Code Directory
# The app/ folder is the core application directory that contains all our handcrafted source code.


bower_components/
# Web Packages Directory
# The bower_components/ folder is the directory where bower web packages are installed as defined in bower.json via bower install.


dist/
#Deployment Directory
# The dist/ folder is the distribution directory where our grunt build process places our application for deployment as defined in Gruntfile.js


node_modules/
# Node Packages Directory
# The node_modules/ folder is where npm packages are installed as defined in package.json via npm install


.gitignore
# .gitignore File
# The .gitignore file specifies intentionally untracked files that Git should ignore during git commits. Files already tracked by Git are not affected.


bower.json
# bower.json File
# The bower.json file defines the web package dependencies that are required by our application and are installed in the bower_components directory via bower install.


firebase.json
# firebase.json File
# The firebase.json file is used by the firebase-tools cli to configure which files are published upon deployment via firebase deploy.


Gruntfile.js
# Gruntfile.js File
# The Gruntfile.js is used to define and configure our core grunt build and deployment tasks including (grunt serve and grunt build). # Development Tasks such as Serve, Watch, and Livereload are defined here. # Build Tasks such as Minify, Concat, Prefix, and etc are defined here.


package.json
# package.json File
# The package.json file defines the application build and deployment dependencies that are required by our application and are installed in the node_modules directory via npm install.


Application Configuration | Overview


/app/app.js

# The app.js file is the main entry point to our backend angularfire application.
# Defines the core angular module (starterlog) for our angular and firebase application.
# Defines the core module dependencies for our angular and firebase application.
# Defines the constants for our application (FIREBASE_URL).
# Defines the ui routes for our application (stateProvider).

Core Module Dependencies:

  • firebase
    This module contains the Firebase client library used by our application and libraries such as AngularFire.
    More at firebase

  • angular-md5
    This module contains the library used by our application to create an MD5 Hash.
    More at angular-md5

  • ui.bootstrap
    This module contains a set of native AngularJS directives based on Bootstrap's markup and CSS.
    As a result no dependency on jQuery or Bootstrap's JavaScript is required.
    The only required dependencies are AngularJS and Boostrap CSS.
    More at ui-bootstrap

  • ui.router
    This module contains the AngularUI Router which is a routing framework for AngularJS.
    The ui-router allows us to congigure our application interface as a state machine defining stateProvider routes.
    More at ui-router

  • ngTable
    This module contains the library that we use to configure and bind all of the datatables in our application.
    More at ng-table

  • textAngular
    This module contains the library that we use to configure rich text editing and wysiwig features in our application.
    More at textAngular
